# БРИФ №2 — «Воздух Алматы» — PWA-дашборд качества воздуха

> Оригинальный бриф владельца. Сохранён в репо как источник истины для всех сессий.

## Роль

Ты — техлид команды, делающей публичный продукт с реальными пользователями, не пет-проект. Владелец — Alexander Kurchakov (Frontend + UI/UX, Almaty; GitHub Alexanderadon). Смог — реальная боль Алматы: продукт полезен по-настоящему, поэтому он лучший кандидат на строчку «мой продукт, есть пользователи» в собесах.

## Зачем (мэппинг на резюме)

- Next.js (ISR/revalidate, RSC) + TypeScript strict — подтверждение ядра резюме
- Работа с внешними API, кэширование, графики — «умеет данные», не только вёрстку
- PWA + Web Push — редкий навык, выделяет среди мидлов
- UI/UX: дата-виз должен быть красивым — это витрина дизайнерской стороны

## Жёсткие правила

- **Только реальные данные.** Источники: OpenAQ API (бесплатный) и/или IQAir free tier; сенсоры Алматы существуют — проверить покрытие через WebSearch до старта. Если станций мало — добавить открытые данные Казгидромета.
- Никакого lorem ipsum, никаких сгенерированных «примерных» значений AQI.
- UI на русском (аудитория — жители Алматы). README RU + краткая EN-секция.
- Не писать «учебный/pet/demo». Это продукт: «Качество воздуха в Алматы в реальном времени».

## Что строим (MVP → v1)

**MVP:** текущий AQI по районам города (карта Leaflet + OpenStreetMap, цветовые зоны) → экран района: график за 24ч/7д/30д → простая шкала «что это значит» (норма/плохо/опасно + рекомендации) → автообновление раз в час (ISR).

**v1:** история в собственной БД (append раз в час), сравнение районов, web-push «воздух стал опасным» (подписка на район), PWA-установка на домашний экран (manifest + SW + иконки), share-карточка района с динамической OG-картинкой.

## Стек

- Next.js 15+ App Router, RSC, ISR (revalidate: 3600), TypeScript strict
- Tailwind CSS v4; графики: Recharts или visx (выбрать по качеству кастомизации, обосновать)
- Leaflet + OSM (без платных ключей карт)
- Neon Postgres (free) + Prisma — история измерений
- Сбор истории: Vercel Cron даёт мало на hobby (2/день) → внешний пинг cron-job.org (free) на /api/collect каждый час с секретом в заголовке
- Web Push: библиотека web-push, VAPID-ключи в env (iOS поддерживает push для установленных PWA)
- Vitest (конвертация AQI, агрегации) + Playwright smoke; GitHub Actions CI
- Деплой: Vercel hobby + Vercel Analytics

## Команда агентов — обязательное разделение

1. **Data-агент** — исследует источники API (какие станции реально живы в Алматы), пишет слой lib/sources с нормализацией и фолбэком между провайдерами. Первым делом, до UI.
2. **Architect** — схема БД истории, стратегия кэширования, контракт /api/collect.
3. **UI/UX-агент** — дизайн-токены, цветовая шкала AQI (доступная для дальтоников — проверить контрасты), скетчи экранов до вёрстки.
4. **Feature-агенты параллельно:** карта / графики+история / push+PWA. Изолированные worktree.
5. **Reviewer (adversarial)** — /code-review high после каждой фазы; отдельно атакует: API-источник лёг, пустая история, отказ от гео/пуш-разрешений.
6. **QA + Tech-writer** — Playwright, Lighthouse, README со скриншотами в конце.
