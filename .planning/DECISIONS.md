# Архитектурные решения — 2026-07-14

## D1. Источники данных: трёхслойная схема

1. **Open-Meteo (модель CAMS)** — нулевой слой, работает без ключей → приложение показывает реальные данные сразу после деплоя. Честная маркировка «модель CAMS (Copernicus), сетка ~40 км».
2. **OpenAQ v3** — станционный слой (124 сенсора AirGradient), включается env-переменной `OPENAQ_API_KEY`. PM2.5 only.
3. **WAQI** — официальный слой (Казгидромет kz-hydromet + US Embassy), включается `WAQI_TOKEN`.

Провайдеры независимы, каждый может упасть/быть не настроен — агрегатор собирает что есть и репортит статусы. AQI считаем сами из концентраций PM (шкала US EPA, ревизия 2024) — не доверяем чужим композитным AQI.

**Отклонение от брифа:** AirKaz исключён (сеть мертва, API нет, копирайт запрещает). IQAir исключён (free tier без концентраций). Вместо них OpenAQ (куда публикует сеть-преемник Almaty Air Initiative) + WAQI.

## D2. Районы: статичный GeoJSON из OSM

8 районов, генерация один раз через Overpass→osmtogeojson→mapshaper (~13 КБ), в бандле. Привязка сенсоров к районам — point-in-polygon на сервере. Слаги: alatau, almaly, auezov, bostandyk, zhetysu, medeu, nauryzbay, turksib. Атрибуция «© участники OpenStreetMap».

## D3. Графики: visx (не Recharts)

Требование брифа — красивый кастомный дата-виз. visx: ~30-40 КБ, полный контроль SVG (градиенты, цветные зоны порогов, кастомные тултипы), статика в RSC. Recharts: 145 КБ + Redux внутри. Обоснование в RESEARCH.md.

## D4. PWA: Serwist; Push: web-push + VAPID

next-pwa мёртв. Serwist — актуальный преемник, назван в официальном гайде Next.js. iOS-ограничения пуша (16.4+, standalone, юзер-жест) — учесть в UI подписки. Фаза 4.

## D5. Шкала AQI

US EPA AQI (ревизия PM2.5 2024 года: Good 0–9.0 мкг/м³ и т.д.), piecewise-linear, правила усечения EPA. 6 категорий с русскими названиями и рекомендациями. Палитра — не дефолтная EPA (красный/зелёный неразличимы при дейтеранопии): подобрать colorblind-safe с прогрессией светлоты, проверить контрасты WCAG.

## D6. БД истории: Supabase Postgres + Prisma (отклонение от брифа: было Neon)

**Итог реализации (2026-07-14):** у аккаунта исчерпан лимит 2 бесплатных Supabase-проектов, поэтому история живёт в изолированной схеме `almaty_air` внутри существующего проекта resto-miniapp (`mdhifvjxrbkzqyuqajzk`, eu-central-1). Доступ — выделенная роль `almaty_air_app` с правами только на эту схему; PostgREST схему не экспонирует. Если владелец освободит слот (удалит спящий «AdamDimErt's Project») — можно мигрировать в отдельный проект.

## D10. Крон сбора: GitHub Actions вместо cron-job.org (отклонение от брифа)

`collect.yml` по расписанию `7 * * * *` дергает `/api/collect` с секретом из repo secrets. Причина: cron-job.org требует отдельной регистрации (шаг владельца), GH Actions на публичном репо бесплатен и уже под рукой. Минус: расписание Actions может опаздывать на минуты — для почасового сбора несущественно.

Причина: у владельца уже есть аккаунт Supabase (инфра resto-miniapp) и подключённый MCP — проект создаётся без единого ручного шага; Neon потребовал бы новой регистрации. Free tier Postgres 500 МБ достаточно (замеры 8 районов × 24 ч × 365 дней — мегабайты). Prisma остаётся. Финальное подтверждение — при старте фазы 3.

## D7. Кэширование

Страницы: ISR `revalidate = 3600` (по брифу). Данные: fetch cache — текущие значения 1800 с, история 3600 с. Лимиты источников при этом не напрягаются (Open-Meteo 10k/день, OpenAQ 2k/час).

## D8. Шаги, которые может сделать только владелец

1. **OpenAQ API key** — регистрация https://explore.openaq.org/register → `OPENAQ_API_KEY` в env (включает станционный слой, 124 сенсора).
2. **WAQI токен** — форма https://aqicn.org/data-platform/token/ → `WAQI_TOKEN` в env (включает Казгидромет).
3. **Письмо команде WAQI** перед публичным запуском (требование их ToS для некоммерческого публичного использования).
4. `vercel login` — только если деплой через MCP не сработает.
5. Финальная проверка PWA на телефоне (DoD).

Без шагов 1–2 приложение полностью рабочее на модельном слое Open-Meteo.

## D9. Ограничения честности данных (запрещено нарушать)

- Модельные данные всегда помечены как модельные.
- Никаких выдуманных значений; при отсутствии данных — честné empty-state.
- us_aqi Open-Meteo не выдавать за PM2.5 AQI (он композитный, бывает озоновым).
- Приложение остаётся бесплатным и без рекламы (условие free tier Open-Meteo и WAQI).
